ZyXEL-NSA310固件分析

00-pre

  • 固件：https://drivers.softpedia.com/get/FIRMWARE/Zyxel/ZyXEL-NSA310-Media-Server-Firmware-470AFK1C0.shtml
  • binwalk

01-发现规律

  • hexdump发现规律：1蓝色 + 1蓝色 + 4红色 + data绿色（data长度由4红色指定，小端

  • 不考虑4字节data，整理成表，进一步观察（未识别ascii的便0x形式，并且也小端表示）

02-推测/验证

  • 0002、0102、0402处，其值类似hash值
  • xx10、xxA0以及ffff共9处，其值类似文件中地址/偏移
  • 加上最开始的0x30偏移，共组成10部分：头部信息、上面的规律字典、4文件的开始及结束

  • 验证hash与文件偏移的对应
  • 共4个文件，而hash值只有3个，为什么数量没对应上？

03-文件1-uImage

1-获取uImage

  • dd提取：dd if=470AFK1C0.bin skip=317 count=5696776 bs=1 of=uimage.bin

  • uImage文件是一个启动镜像（boot image），其内包含内核及RAM（Linux kernel and a RAM disk）

2-获取zImage

  • 有现成工具extract_uImage.sh，可用于提取（随便Google即可https://gist.github.com/adamvr/1079762
  • zImage用来启动Linux，其包含了Linux内核和RAM disk，RAM就是设备上根目录

3-自动提取

4-手动提取

  • dd + gunzip得到more.bin.gz
  • dd+gunzip得到initrd.bin
  • cpio挂载
  • 完整流程
    • firmware.bin
    • （dd）uImage.bin：启动镜像（boot image），其内包含内核及RAM（Linux kernel and a RAM disk）
    • （extract_uimage.sh脚本）zImage：用来启动Linux，其包含了Linux内核和RAM disk，RAM就是设备上根目录
    • （dd+gunzip）more.bin
    • （dd+gunzip）initrd.bin：cpio格式的ram disk
    • cpio挂载
